## Failed Tests / Bugs

### BUG-1: --min-confidence HIGH returns empty when data exists (MEDIUM)

**Command:** `/edge --min-confidence HIGH --json`
**Output:** `{"ok":true,"command":"edge","data":[]}`

But `/edge --min-edge 5 --json` returns many "very_high" and "high" confidence rows.

**Root cause:** In `edge.ts` line 20-21, when `--min-confidence` is set, it calls `getActionableEdges(db, args.minConfidence)`. The value passed is `"HIGH"` (uppercase from CLI arg) but the database stores confidence as lowercase (`"high"`, `"very_high"`). The SQL query likely does a case-sensitive comparison.

Additionally, `--min-confidence HIGH` should also return `very_high` results (since very_high > high), but `getActionableEdges` may filter for exact match rather than "at least this level".

**Fix needed:** Normalize to lowercase in parse-args or edge.ts, and implement hierarchical filtering (LOW returns all, MODERATE returns moderate+high+very_high, etc.)

### BUG-2: resolveMarket() with series ticker (KXBTC) hangs >30s (MEDIUM)

**Command:** `/analyze KXBTC --json`
**Result:** Command does not return within 30 seconds (times out)

**Root cause:** The series ticker resolution in `resolveMarket()` calls `GET /events?series_ticker=KXBTC&status=open&with_nested_markets=true&limit=100`. KXBTC is a very active series with many events, and fetching all with nested markets is slow. After resolution, the Octagon API call for the highest-volume market may also be slow.

**Fix needed:** Consider adding a timeout wrapper around resolveMarket, or reducing the limit for series ticker lookups.

### BUG-3: --refresh flag times out on Octagon API (LOW)

**Command:** `/analyze KXBTCMAX100-26-MAR --refresh --json`
**Result:** Does not return within 30 seconds

This is likely an Octagon API latency issue rather than a code bug. The `refresh` variant triggers a fresh Octagon report generation which can take 30+ seconds. The non-refresh cache path works fine (returns in ~2s).

**Possible fix:** Add a CLI-level timeout with a helpful message like "Octagon report generation is taking longer than expected. Try again later or use cached data."

## Discrepancies from Doc

### 1. Edge table has no color coding
The edge table is rendered as plain ASCII -- no ANSI color codes for positive/negative edges or confidence levels. If the v2.0 doc specifies color coding, this is missing. The table structure (box-drawing, columns) is clean but monochrome.

### 2. Liquidity check produces NaN for dollar-format prices
The risk gate liquidity check computes `spread = market.yes_ask - market.yes_bid` using cent fields, but the Kalshi API now returns dollar-string fields (`yes_ask_dollars`). The analyze command's `parseMarketProb` handles both formats, but `riskGate` does not -- it accesses `market.yes_ask` and `market.yes_bid` directly, which may be NaN when only dollar fields are present.

**Evidence:** `"reason": "Spread NaN cents >= max 5 cents"`

### 3. --min-confidence case sensitivity
The doc likely expects case-insensitive matching (`HIGH`, `high`, `High` should all work), but the implementation appears case-sensitive.

## Recommended Fixes

### Priority 1 (Bugs affecting core functionality)

1. **Fix --min-confidence filtering** (`src/commands/edge.ts` + `src/db/edge.ts`):
   - Normalize input to lowercase: `args.minConfidence?.toLowerCase()`
   - Implement hierarchical filtering: if user passes "high", return high + very_high
   - Add a confidence level ordering: `{low: 0, moderate: 1, high: 2, very_high: 3}`

2. **Fix risk gate NaN spread** (`src/risk/gate.ts`):
   - Use the same `parsePriceField()` helper from analyze.ts to handle both dollar-string and cent fields
   - Import or duplicate the price parsing logic so spread calculation works with both API formats

### Priority 2 (Performance / UX)

3. **Add timeout for series ticker resolution** (`src/commands/analyze.ts`):
   - Wrap `resolveMarket()` in a Promise.race with a 15s timeout
   - Return a helpful error: "Series ticker KXBTC has too many active events. Try a specific event ticker like KXBTC-26MAR31."

4. **Add timeout for --refresh Octagon calls** (`src/scan/octagon-client.ts`):
   - Add a configurable timeout (default 60s) for refresh API calls
   - Return cached data as fallback with a warning message

### Priority 3 (Polish)

5. **Add ANSI color coding to edge table** (`src/commands/scan-formatters.ts`):
   - Green for positive edge, red for negative
   - Bold for very_high confidence
   - Muted/dim for low confidence

6. **Normalize confidence input across all entry points**:
   - Accept case-insensitive values everywhere
   - Add validation with helpful error for invalid confidence levels
